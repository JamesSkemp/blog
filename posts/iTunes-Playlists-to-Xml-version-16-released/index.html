iTunes Playlists to Xml - version 1.6 released

Version 1.6 of iTunes Playlists to Xml is now available for download.

Download iTunes Playlists to Xml 1.6.

Installing and more information

If you're new to iTunes Playlists to Xml, I've created a 'product page' for iTunes Playlists to Xml with all the information needed to get started.

Upgrading

To upgrade from any version of 1.4 or 1.5 you need to replace the following files with those in the above archive:

  • AlbumName.xslt
  • iTunesPlaylists2Xml.xslt
  • iTunesPlaylistsToXml.exe
  • NameArtistAlbumPlay.xslt

Your configuration file should be upgraded with the new functionality automatically.

To upgrade from version 1.3 or earlier, extract the contents of the zip replacing everything but the iTunesPlaylistsToXml.exe.config file; when you start the application after upgrading the configuration file should be automatically upgraded with all new settings. If you get a configuration error, then replace this file with the new version. Remember to update your name in the Settings menu.

Changes from version 1.5.3

With the release of .NET Framework 4 a few weeks ago, I figure it's safe to move from .NET Framework 2.0 to 3.5 in iTunes Playlists to Xml. Therefore, .NET Framework 3.5 is now required.

In addition to the new requirement, version 1.6 of iTunes Playlists to Xml adds the ability to save playlists to HTML, in addition to XML.

Features for future releases

A future release will allow you to change the elements that are output by default via the user interface (it's possible now via the command line).
