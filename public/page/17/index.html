The benefits of using a file for Log Parser SQL queries
One of the things I always forget about when I'm using Microsoft Log Parser is that you can define your SQL query in a file and run it, instead of putting it in the command line. For example, the following will run whatever SQL is defined in strivinglife.robots.sql. logparser -i:w3c -o:xml file:strivinglife.robots.sql Where this comes in handy is for something like the Log Parser query I defined today, that parses IIS logs for requests to the robots.txt file and outputs the ip address and user-agent of the request, with a total count from that combination.
